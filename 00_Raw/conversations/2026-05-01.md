### 2. 초기 UI/UX 와이어프레임 방향 설정 (MVP Focus)

**목표:** 사용자가 사진 업로드부터 스타일 추천 결과 확인까지의 과정을 **최소한의 단계(Frictionless)**로 완료하도록 설계합니다.

#### A. 핵심 사용자 플로우 (User Flow)
1.  **Onboarding/Entry:** 서비스 접속 → 가입/로그인 (선택 사항 최소화)
2.  **Input Stage:** 얼굴 사진 업로드 및 기본 정보 입력
3.  **Analysis Stage:** AI 분석 진행 (로딩 상태 명확히 표시)
4.  **Result Stage:** 맞춤 스타일 추천 결과 제시
5.  **Action Stage:** 추천 스타일 상세 보기 및 저장/다음 단계 이동

#### B. 핵심 화면 구성 (Wireframe Direction)

#### A. 사용자 인증 및 프로필 관리 (Authentication & Profile)
1.  **회원가입/로그인:** 이메일/소셜 로그인 연동 (선택 사항).
2.  **사용자 프로필 저장:** 이름, 선호 스타일, 이전 분석 기록 등 개인화된 데이터 저장 기능.
3.  **스타일 프리셋 관리:** 사용자가 저장하거나 추천받은 스타일(예: 빈티지, 모던, 시크)의 메타데이터 저장.

#### B. 핵심 AI 추천 기능 (Core Recommendation Flow)
1.  **이미지 업로드 인터페이스:** 사용자가 얼굴 사진을 업로드할 수 있는 기능.
2.  **AI 분석 API 호출:** 업로드된 이미지를 백엔드(Python/ML 모델)로 전송하여 얼굴형 및 이목구비 데이터를 추출하도록 요청.
3.  **스타일 추천 엔진 실행:** 추출된 데이터와 사용자의 선호도 데이터를 기반으로 스타일을 추천하는 로직 실행.
4.  **결과 시각화:** 추천된 스타일과 관련된 이미지 예시, 상세 설명(Designer에서 정의한 톤앤매너 반영)을 사용자에게 제시.

#### C. 콘텐츠 및 관리 기능 (Content & Management)
1.  **스타일 데이터베이스 연동:** 미리 정의된 스타일 가이드(DB)를 검색하고 필터링하여 제공.
2.  **결과 저장:** 사용자가 마음에 드는 추천 결과를 저장하고 북마크 할 수 있는 기능.

### 3. 데이터베이스 구조 초안 (Schema Outline - PostgreSQL)

MVP에 필요한 최소한의 테이블 정의입니다. AI 모델 자체의 복잡한 학습 데이터는 별도의 데이터 레이크/저장소(S3 등)에 보관하고, DB는 서비스 운영에 필요한 메타데이터 및 사용자 관계 관리에 집중합니다.
