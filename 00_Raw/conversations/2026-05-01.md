### 2. 초기 UI/UX 와이어프레임 방향 설정 (MVP Focus)

**목표:** 사용자가 사진 업로드부터 스타일 추천 결과 확인까지의 과정을 **최소한의 단계(Frictionless)**로 완료하도록 설계합니다.

#### A. 핵심 사용자 플로우 (User Flow)
1.  **Onboarding/Entry:** 서비스 접속 → 가입/로그인 (선택 사항 최소화)
2.  **Input Stage:** 얼굴 사진 업로드 및 기본 정보 입력
3.  **Analysis Stage:** AI 분석 진행 (로딩 상태 명확히 표시)
4.  **Result Stage:** 맞춤 스타일 추천 결과 제시
5.  **Action Stage:** 추천 스타일 상세 보기 및 저장/다음 단계 이동

#### B. 핵심 화면 구성 (Wireframe Direction)

#### A. 사용자 인증 및 프로필 관리 (Authentication & Profile)
1.  **회원가입/로그인:** 이메일/소셜 로그인 연동 (선택 사항).
2.  **사용자 프로필 저장:** 이름, 선호 스타일, 이전 분석 기록 등 개인화된 데이터 저장 기능.
3.  **스타일 프리셋 관리:** 사용자가 저장하거나 추천받은 스타일(예: 빈티지, 모던, 시크)의 메타데이터 저장.

#### B. 핵심 AI 추천 기능 (Core Recommendation Flow)
1.  **이미지 업로드 인터페이스:** 사용자가 얼굴 사진을 업로드할 수 있는 기능.
2.  **AI 분석 API 호출:** 업로드된 이미지를 백엔드(Python/ML 모델)로 전송하여 얼굴형 및 이목구비 데이터를 추출하도록 요청.
3.  **스타일 추천 엔진 실행:** 추출된 데이터와 사용자의 선호도 데이터를 기반으로 스타일을 추천하는 로직 실행.
4.  **결과 시각화:** 추천된 스타일과 관련된 이미지 예시, 상세 설명(Designer에서 정의한 톤앤매너 반영)을 사용자에게 제시.

#### C. 콘텐츠 및 관리 기능 (Content & Management)
1.  **스타일 데이터베이스 연동:** 미리 정의된 스타일 가이드(DB)를 검색하고 필터링하여 제공.
2.  **결과 저장:** 사용자가 마음에 드는 추천 결과를 저장하고 북마크 할 수 있는 기능.

### 3. 데이터베이스 구조 초안 (Schema Outline - PostgreSQL)

MVP에 필요한 최소한의 테이블 정의입니다. AI 모델 자체의 복잡한 학습 데이터는 별도의 데이터 레이크/저장소(S3 등)에 보관하고, DB는 서비스 운영에 필요한 메타데이터 및 사용자 관계 관리에 집중합니다.

### 3. 사용자 Pain Points (가장 흔한 고민 데이터)

2.  **"관리의 어려움" (Maintenance Struggle):** 예쁘게 자른 후 집에서 일상적으로 스타일을 유지하고 관리하는 방법을 모름. (잦은 미용실 방문에 대한 부담)
3.  **"트렌드와의 괴리감" (Trend Gap):** SNS에서 유행하는 스타일을 따라 하고 싶지만, 내 라이프스타일이나 얼굴에 맞지 않을까 봐 망설임.
4.  **"손상에 대한 불안감" (Damage Anxiety):** 원하는 스타일링/염색 과정에서 모발이 손상될까 봐 걱정함.

### 2. 각 주제별 실행 전략 및 근거 (Actionable Steps)

#### 🥇 우선순위 1: 초정밀 진단 기반의 '퍼스널 스타일 포트폴리오' 구축
*   **실행 방안:** 단순한 '추천'을 넘어, 사용자의 얼굴형, 피부 톤, 라이프스타일(직업, 취미), 선호하는 관리 난이도 등을 종합하여 **‘나만의 스타일 DNA’**를 생성합니다. 이 DNA를 기반으로 3~5가지의 구체적인 스타일 옵션(A/B/C안)을 제시하고, 각 옵션별 예상 투자 비용과 유지 노력을 명시해야 합니다.
*   **수익화 근거:** 사용자가 '결정의 피로감'을 느끼는 지점에서, AI가 제공하는 **'최종 결정 지원 리포트'**에 프리미엄 가격을 책정합니다. (정보의 희소성과 의사결정 비용 절감)

#### 🥈 우선순위 2: 스타일 유지 관리 및 비용 효율성 가이드
*   **실행 방안:** 진단된 스타일에 맞춰 필요한 제품(헤어케어 제품, 도구 등) 추천 리스트를 제공하고, 각 항목에 대한 **최소 투자 금액 대비 최대 효과(ROI)**를 시각적으로 제시합니다. (예: "손상 최소화 펌을 위한 필수 관리 용품 세트")
*   **수익화 근거:** 사용자들은 스타일 결정 후 '유지 비용'에 대한 불안감을 느낍니다. 엔진이 단순 추천에서 **'스타일 유지 예산 관리 솔루션'**으로 확장되어, 제휴된 제품 구매를 유도함으로써 커미션을 확보합니다.

#### 🥉 우선순위 3: 트렌드 예측 기반의 선제적 스타일 업그레이드
*   **실행 방안:** 시장 검색 트렌드(Researcher 결과)와 연결하여, "다음 시즌 유행할 스타일", "나에게 가장 적합한 다음 변화" 등의 콘텐츠를 주기적으로 발행합니다. 이는 단순한 '현재'가 아닌 **'미래의 나'에 대한 투자**라는 프레임을 제공해야 합니다.
*   **수익화 근거:** 트렌드 예측 정보를 구독자 전용으로 제공하거나, 특정 트렌드를 반영한 **'업그레이드 패키지'**를 판매하여 시의적절한 소비를 유도합니다.

#### 🏅 우선순위 4: 스타일과 라이프스타일의 통합 시뮬레이션
*   **실행 방안:** 사용자의 직장 문화, 주말 활동, 선호하는 복장 등을 입력받아 "이 스타일이 당신의 일상에 얼마나 잘 어울릴지"를 스토리텔링 방식으로 보여줍니다. (예: "회의 중 자신감 표현하기", "주말 데이트 룩 완성하기")
*   **수익화 근거:** 이 기능은 **'경험 판매(Experience Selling)'**의 영역입니다. 시뮬레이션 결과에 따라 맞춤형 코디 제안이나 관련 라이프스타일 제품을 연결하는 인터페이스를 통해 높은 전환율을 확보합니다.

#### 🎖️ 우선순위 5: 전문가 검증 기반의 '맞춤형 스타일 컨설팅' 연계
*   **실행 방안:** AI 진단 결과를 바탕으로, 사용자가 원하는 특정 목표(예: '손상 없이 볼륨 살리기')를 설정하면, 해당 분야 전문 디자이너/교육 전문가와 **AI가 선별한 최적의 매칭 시스템**을 통해 1:1 컨설팅을 예약하게 합니다.
*   **수익화 근거:** AI는 대량의 진단을 처리하고, 인간 전문가는 고부가가치 '최종 검증 및 실행' 역할을 수행합니다. Midmi는 이 연결고리에서 **중개 수수료(Commission)**를 확보하여 플랫폼 가치를 극대화합니다.

## 1. TypeScript Data Schema (`types/style.ts`)

데이터의 무결성을 위해 `enum`과 `interface`를 사용하여 엄격하게 정의합니다.

```typescript
// 1. 기초 데이터 타입 정의
export type FaceShape = 'oval' | 'round' | 'angular' | 'long' | 'heart' | 'inverted-triangle';
export type SkinToneType = 'warm' | 'cool' | 'neutral';
export type SkinBrightness = 'light' | 'medium' | 'dark';
export type HairThickness = 'fine' | 'medium' | 'thick';
export type CurlPattern = 'straight' | 'wavy' | 'curly' | 'coily';
export type HairDensity = 'low' | 'medium' | 'high';
export type HairDamage = 'healthy' | 'dry' | 'damaged';
export type Vibe = 'lovely' | 'chic' | 'casual' | 'formal' | 'hip' | 'classic';
export type MaintenanceLevel = 'low' | 'medium' | 'high';

// 2. 입력 데이터 스키마 (User Input)
export interface UserStyleInput {
  face: {
    shape: FaceShape;
    skinTone: {
      type: SkinToneType;
      brightness: SkinBrightness;
    };
    features: {
      eyeFocus: string; // 예: "deep eyes"
      noseHeight: string; // 예: "high bridge"
      lipThickness: string; // 예: "full lips"
    };
  };
  hair: {
    thickness: HairThickness;
    pattern: CurlPattern;
    density: HairDensity;
    damage: HairDamage;
  };
  lifestyle: {
    desiredVibe: Vibe[];
    maintenance: MaintenanceLevel;
    environment: string; // 예: "office", "artistic"
  };
  accessories: {
    hasGlasses: boolean;
    fashionStyle: string[];
  };
}

// 3. AI 분석 결과 스키마 (AI Response)
export interface StyleAnalysisResponse {
  userId: string;
  timestamp: string;
  summary: {
    keywords: string[];
    score: number; // 0.0 ~ 5.0
    coreIdentity: string;
  };
  detailedAnalysis: {
    faceAnalysis: string;
    hairAnalysis: string;
    lifestyleCompatibility: string;
  };
  recommendations: {
    hairStyles: Array<{
      name: string;
      reason: string;
      imagePrompt: string; // 이미지 생성 AI(DALL-E/Midjourney)용 프롬프트
    }>;
    hairColors: Array<{
      colorName: string;
      reason: string;
    }>;
    maintenanceTips: string[];
  };
}
```
