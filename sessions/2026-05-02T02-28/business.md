## 1. 데이터 유출의 비즈니스 임팩트 분석

데이터 유출은 단순한 기술적 사고를 넘어, 미드미의 비즈니스 모델 자체를 붕괴시적 위험을 내포합니다.

### 📉 브랜드 신뢰도(Brand Trust) 측면
- **브랜드 정체성 훼손:** '친절하고 전문적인' 브랜드 톤이 '무책임하고 위험한' 이미지로 즉각 전환.
- **바이럴 역효과:** AI 서비스 특성상 SNS 공유가 핵심 마케팅인데, 보안 사고 발생 시 부정적 바이럴이 신규 고객 획득 비용(CAC)을 기하급수적으로 상승시킴.

### 📉 LTV(고객 생애 가치) 및 재무적 측면
- **Churn Rate(이탈률) 급증:** 민감 정보(얼굴 사진) 유출 시, 기존 사용자의 재방문율이 0에 수렴하며 LTV가 급락함.
- **LTV 하락 메커니즘:** `LTV = (평균 결제 금액 × 구매 빈도) × 고객 유지 기간`. 유출 사고는 '고객 유지 기간'을 강제로 종료시킴.
- **직접적 비용 발생:** PIPA(개인정보보호법) 위반에 따른 과징금(매출액의 일정 비율), 집단 소송 비용, 시스템 복구 및 보안 감사 비용 발생.

## 2. 보안 수준별 비용-리스크 관리 매트릭스

모든 데이터에 최고 수준의 보안을 적용하는 것은 비즈니스적으로 비효율적입니다. 데이터 중요도에 따라 자원을 차등 배분합니다.

| 데이터 등급 | 대상 데이터 | 보안 수준 (Target) | 예상 보안 비용 | 리스크 수준 | 전략적 대응 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Level 1 (Low)** | 이메일, 이름, 선호 스타일 | 표준 암호화, 접근 제어 | **저(Low)** | **중(Medium)** | 일반적인 웹 보안 표준(SSL/TLS) 준수 |
| **Level 2 (Mid)** | 진단 결과 데이터, 활동 로그 | 데이터 비식별화, 로그 모니터링 | **중(Mid)** | **고(High)** | 데이터베이스 접근 권한 최소화(RBAC) |
| **Level 3 (Critical)** | **얼굴 원본 이미지, 생체 특징점** | **분리 저장, 별도 동의, 2FA, 암호화** | **고(High)** | **극고(Extreme)** | **PIPA 준수 필수, 물리적/논리적 격리** |

## 3. 단계별 보안 운영 로드맵 (Decision Recommendation)

초기 리소스가 제한된 MVP 단계에서는 **'Compliance-First(법적 준수 우선)'** 전략을 채택하고, 성장에 따라 **'Advanced Defense(고급 방어)'**로 확장할 것을 권고합니다(A/B 선택지 중 A안 채택).

### [A안] 단계적 보안 강화 전략 (권고)
- **Phase 1: MVP 단계 (Compliance Focus)**
    - **목표:** 법적 리스크 제거 및 최소 비용 운영.
    - **핵심 액션:** PIPA 필수 항목(민감정보 별도 동의) 준수, Level 3 데이터의 암호화 저장, 관리자 페이지 2FA 적용.
    - **비용:** 개발 초기 인프라 구축 비용에 포함 가능 수준.
- **Phase 2: Scale-up 단계 (Proactive Defense)**
    - **목표:** AI 특화 공격(Model Inversion 등) 방어 및 브랜드 신뢰도 자산화.
    - **핵심 액션:** 데이터 비식별화(Anonymization) 자동화, 정기적 보안 취약점 점검(Vulnerability Scanning), 보안 인증(ISMS 등) 준비.
    - **비용:** 보안 전문 인력 또는 외부 보안 솔루션 도입 필요.

### [B안] 전방위적 최고 수준 보안 적용 (비권고)
- **내용:** 모든 데이터에 Level 3 보안 적용.
- **리스러:** 초기 개발 속도(Time-to-Market) 급감 및 인프라 비용 과다로 인한 수익화 모델 검증 실패 위험.
