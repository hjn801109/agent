# 📋 작업 브리프

**원 명령:** 한번에 글자수를 5000자가 넘으면 안되

## 요약
출력 글자 수 5000자 제한 규칙을 확립하고, 이를 시스템 운영 및 콘텐츠 생성 프로세스에 반영하기 위한 작업입니다.

## 분배
- **📱 Secretary**: 새로운 운영 규칙(한 번에 5000자 제한)을 업무 가이드라인 및 매뉴얼에 즉시 업데이트하고, 모든 에이전트가 인지할 수 있도록 공지 요약본을 작성하세요.
- **💻 Developer**: 텍스트 생성 및 데이터 처리 시 5000자 초과 여부를 체크하고, 초과 시 데이터를 자동으로 분할(Chunking)하거나 사용자에게 알림을 보내는 로직을 검토하여 반영하세요.
- **✍️ Writer**: 향후 모든 스크립트 및 문서 작성 시 5000자 제한을 준수할 수 있도록, 긴 분량의 콘텐츠는 파트별(Part 1, Part 2 등)로 나누어 구성하는 작성 전략을 수립하세요.
