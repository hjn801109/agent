**원 명령:** 이제부터 작업할때는 ### 2. "모듈화/분할 작성" 전략 (가장 권장하는 방법)
한 번에 전체 파일을 다 만들려고 하지 말고, 파일의 구조(목차)를 먼저 잡은 뒤 섹션별로 나누어 요청하세요.
*   1단계: "전체 파일의 목차와 구조를 먼저 짜줘."
*   2단계: "이제 목차 중 1. 서론 부분만 상세하게 작성해줘."
*   3단계: "이어서 2. 본론 부분을 작성해줘."
*   장점: 이렇게 하면 각 섹션에 할당되는 토큰 양이 늘어나 훨씬 더 깊이 있고 풍부한 내용을 얻을 수 있습니다.

### 3. "구조적 프롬프팅" 전략 (설계도 제공)
작성할 내용의 '뼈대'를 미리 프롬프트에 포함시키세요.
*   방법: "파일을 작성할 때 아래의 구조를 반드시 지켜줘: [목차 1, 목차 2, 목차 3...]"
*   효과: 모델이 전체 맥락을 인지한 상태에서 작업하기 때문에, 끊겼을 때 다시 시작하더라도 논리적 일관성을 유지하기가 훨씬 쉽습니다.

### 4. "중간 체크포인트" 설정 (검토 병행)
내용이 길어질 경우, 중간에 "여기까지 작성된 내용이 논리적으로 맞는지 확인해줘"라고 요청하여 흐름을 점검하세요.
*   효과: 끊김 현상 발생 시, 모델이 스스로 자신이 어디까지 썼는지 인지하게 만들어 '재시작'의 정확도를 높입니다.이렇게 해주고
