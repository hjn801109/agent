# 📢 [공지] 신규 작업 표준(SOP) 도입: '모듈화 및 구조적 프롬프팅 전략'

CEO의 지시에 따라, 모든 에이전트(Designer, Writer, Developer)는 작업의 품질을 극대화하고 출력 제한 문제를 해결하기 위한 새로운 작업 표준(SOP)을 즉시 적용하십시오.

## 🛠️ 1. 핵심 전략: 모듈화 및 분할 작성 (Modular/Split Writing)
**"한 번에 완성하려 하지 말고, 층을 쌓듯 작성하십시오."**
- **단계별 프로세스:**
    1.  **[1단계: 설계]** 전체 파일의 **목차(T0C)와 구조**를 먼저 생성 요청.
    2.  **[2단계: 상세화]** 확정된 목차 중 **특정 섹션(예: 1. 서론)**만 지정하여 상세 작성 요청.
    3.  **[3단계: 확장]** 순차적으로 다음 섹션을 요청하여 작업 완료.
- **기대 효과:** 섹션별 할당 토큰 양을 극대화하여 내용의 깊이(Depth)와 풍부함 확보.

## 🏗️ 2. 구조적 프롬프팅 (Structural Prompting)
**"설계도를 프롬프트에 포함하여 맥락을 고정하십시오."**
- **실행 방법:** 작업을 요청할 때 반드시 작업할 내용의 **'뼈대(Outline)'**를 프롬프트에 명시.
    - *예시: "아래 구조를 반드시 준수하여 작성할 것: [1. 개요, 2. 기술 스펙, 3. UI 가이드]"*
- **기대 효과:** 작업 중 끊김(Token Limit)이 발생하더라도, 재개 시 논리적 일관성을 즉시 유지 가능.

## 🔍 3. 중간 체크포인트 설정 (Intermediate Checkpoint)
**"중간 검토를 통해 논리의 흐름을 점검하십시오."**
- **실행 방법:** 내용이 길어질 경우, 특정 섹션 완료 시점에 **"여기까지의 내용이 논리적으로 타당한지 확인해줘"**라고 요청.
- **기대 효과:** 오류의 조기 발견 및 모델의 작업 위치(Context Awareness) 인지 정확도 향상.

| :--- | :--- |

| **Designer** | 전체 UI 컴포넌트 구조 설계 $\rightarrow$ 섹션별(Hero, Feature, Footer) 상세 디자인 가이드 작성 |
